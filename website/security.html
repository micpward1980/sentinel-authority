SECURITY ARCHITECTURE

Zero Trust by Design

ENVELO was built so Sentinel Authority never needs access to your systems, data, or network. Every security property is structural, not contractual.

CORE GUARANTEE
What ENVELO cannot do — by architecture, not policy
NO INBOUND PORTS NO REMOTE ACCESS NO DATA COLLECTION NO CLOUD COMMANDS

The agent opens zero listening ports. Sentinel Authority has no mechanism to connect to, access, or control your infrastructure. Communication is strictly one-way: telemetry out, nothing in.

ATTACK SURFACE
Can Sentinel Authority access our systems remotely?
No. ENVELO opens zero inbound ports and accepts zero inbound connections. There is no SSH, no API endpoint, no reverse tunnel. The architecture makes remote access structurally impossible — not just policy-prohibited.
What happens if the connection to Sentinel drops?
The agent enters fail-closed mode: all autonomous actions are blocked until connectivity is restored. This is the safe state. No data is lost — telemetry is queued locally with cryptographic ordering and transmitted when the connection resumes.
Could a compromised Sentinel server send malicious commands?
No. The agent does not accept commands from any external source. It only transmits telemetry outbound. Even if Sentinel's servers were fully compromised, there is no code path to send instructions to the agent.
DATA PRIVACY
What data does ENVELO transmit?
Only ODD parameter values: speed, position, heading, obstacle distance, weather classification — the operational telemetry defined in your boundary specification. No PHI, no PII, no business data, no logs, no screenshots, no video.
Is ENVELO compatible with HIPAA / SOC 2 / ISO 27001?
Yes. Because ENVELO transmits no PHI or PII, it operates outside the scope of HIPAA covered data. For SOC 2 and ISO 27001 environments, the agent's outbound-only architecture, certificate pinning, and cryptographic audit trail satisfy access control and logging requirements. BAA available on request.
NETWORK
What network access does ENVELO require?
Outbound HTTPS only (port 443) to api.sentinelauthority.org. No inbound ports, no UDP, no custom protocols. Compatible with corporate proxies and firewalls — allowlist a single domain.
How is the connection secured?
TLS 1.3 with certificate pinning. The agent pins Sentinel Authority's certificate fingerprint at install time, preventing MITM interception even if a CA is compromised. All telemetry is additionally signed with the agent's private key.
AUDITABILITY
How can we verify what the agent is sending?
Run envelo export to generate an auditor-ready bundle containing every boundary, every telemetry schema, and every enforcement rule. Run envelo events for the full violation and enforcement log.
Is the telemetry tamper-proof?
Yes. Every telemetry packet is cryptographically signed and hash-chain linked to the previous packet. Any gap, reorder, or modification breaks the chain and is flagged automatically. This provides non-repudiable evidence for certification and audit.
