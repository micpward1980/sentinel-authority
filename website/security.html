Security Architecture

Security FAQ for IT & Compliance Teams

Enterprise security teams ask tough questions. Here are the answers, backed by architecture.

The Core Guarantee

Sentinel Authority cannot send commands to your system, execute code on your infrastructure, access your network, or control your autonomous systems. This is architecturally enforced, not policy-based.

✗ No Remote Commands ✗ No Code Execution ✗ No Network Access ✗ No Actuator Control

Attack Surface & Compromise

What if Sentinel Authority is compromised?
There is no path back to your systems. The agent opens no listening ports and accepts no inbound connections. Even a fully compromised Sentinel Authority cannot send commands, push code, or access your infrastructure. The one-way architecture makes this physically impossible.
Can Sentinel push updates or execute code on our systems?
No. All updates are customer-initiated. We cannot remotely modify, patch, or execute anything on your infrastructure. The agent fetches boundary configurations (JSON data only) but never accepts executable code.
Can the agent access our application data?
No. ENVELO validates action boundaries only. It cannot read, store, or transmit your application data, databases, or files. It sees only the parameters you explicitly pass to boundary checks (e.g., "speed=50").

Data Privacy & HIPAA

Does ENVELO transmit PHI or PII?
No. Telemetry contains only operational metrics—boundary proximity, enforcement events, timestamps. No patient data, no payload content, no PII ever leaves your system. You control what parameters are passed to the agent.
Is a BAA (Business Associate Agreement) available?
Yes. Business Associate Agreements are available for healthcare customers upon request. Contact conformance@sentinelauthority.org.

Network & Firewall

What firewall rules are required?
Outbound HTTPS (port 443) to api.sentinelauthority.org only. No inbound ports. No DMZ exposure. No VPN tunnels. The agent initiates all connections.
What if Sentinel Authority goes offline?
ENVELO continues enforcing locally using cached boundaries. Enforcement decisions never depend on external connectivity. Your system remains protected. Telemetry is buffered and synced when connectivity returns.

Auditability & Compliance

Is the agent code auditable?
Yes. Enterprise customers receive full source access for security review prior to deployment. We welcome third-party security audits.
What compliance certifications do you have?
SOC 2 Type II in progress. Architecture designed to exceed SOC 2 requirements. Annual third-party penetration testing. Contact us for the latest compliance documentation.
