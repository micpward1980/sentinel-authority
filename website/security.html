Security Architecture

Zero Trust, Zero Access

How ENVELO Agent protects your systems

Enterprise security teams ask: "Is this agent a backdoor into our systems?" The answer is architecturally impossible. The ENVELO Agent is designed with a one-way data flow that makes remote access or control physically impossible.

The Core Guarantee

Sentinel Authority cannot send commands to your system, execute code on your infrastructure, access your network, or control your autonomous systems. The agent only observes and reports—it has no actuator access.

One-Way Data Architecture

The ENVELO Agent uses an outbound-only communication model. All connections are initiated by the agent on your infrastructure. There are no listening ports, no inbound connections, and no way for external parties to "reach in."

Security Controls

No Listening Ports

The agent opens zero inbound ports. It only makes outbound HTTPS connections to api.sentinelauthority.org.

Customer-Controlled

You deploy, configure, and can terminate the agent at any time. We cannot start, stop, or modify it remotely.

TLS 1.3 Encryption

All communications use TLS 1.3 with certificate pinning. Data is encrypted in transit and at rest.

API Key Rotation

Rotate credentials anytime from your dashboard. Revoke access instantly if needed.

Auditable Source

The agent source code is available for security review before deployment. No obfuscation, no hidden functionality.

Data Minimization

We collect only boundary evaluation results—not your algorithms, training data, or proprietary logic.

Network Security

Firewall Configuration

The agent requires only one outbound rule:

  • Destination: api.sentinelauthority.org
  • Port: 443 (HTTPS)
  • Direction: Outbound only
  • Protocol: TLS 1.3

No inbound firewall rules are required. The agent does not accept incoming connections.

Network Isolation

For maximum security, deploy the agent in a network segment with:

  • Read-only access to the ENVELO interlock's decision log
  • Outbound HTTPS to api.sentinelauthority.org
  • No other network access

Important Clarification

The ENVELO Agent does not implement your safety interlock—your system does. The agent only observes and reports on decisions your interlock has already made. Even if the agent were compromised, it has no pathway to affect your system's behavior.
