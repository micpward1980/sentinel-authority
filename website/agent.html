ENVELO Agent

Enforced Non-Violable Execution-Limit Override. One command installs. One command starts. The agent does everything.

Quick Start v2.0

Install & Deploy

Open a terminal on the target system and run:

curl -sSL https://get.sentinelauthority.org | bash

The installer prompts for your Certificate ID and API Key (both visible in the portal under ENVELO Agent tab). Then it automatically downloads boundaries, scans for telemetry sources, and configures everything.

After Install
1

Validate

envelo validate

Pre-flight check: credentials, API connectivity, source mappings, fail-closed mode.

2

Start

envelo start -d

Starts the enforcement agent as a background daemon. For auto-start on boot: envelo service install

3

Begin CAT-72

Go to app.sentinelauthority.org → your case → CAT-72 tab → click Begin Test. The 72-hour timer starts immediately.

Every operation — monitoring, diagnostics, infrastructure, boundaries — is a single command.

Security Model

Zero Inbound. Zero Trust.

  • Outbound only — ENVELO opens zero inbound ports on your network
  • No remote access — Sentinel cannot access your system, network, or data
  • No commands from cloud — The connection is one-way: telemetry out, nothing in
  • No PHI/PII — Only ODD parameter values are transmitted. No business data.
  • Signed telemetry — Every packet is cryptographically signed and hash-chain linked
  • Fail-closed — If the connection drops, the agent blocks all actions (safe state)
  • Certificate pinning — TLS 1.3 with pinning prevents man-in-the-middle interception
