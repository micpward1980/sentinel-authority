RUNTIME ENFORCEMENT

ENVELO Agent

Enforced Non-Violable Execution-Limit Override. One command installs. One command starts. The agent does everything.

QUICK START V2.0

Open a terminal on the target system and run:

curl -sSL https://get.sentinelauthority.org | bash

The installer prompts for your Certificate ID and API Key (both visible in the portal under ENVELO Agent tab). Automatically downloads boundaries, scans for telemetry sources, and configures everything.

AFTER INSTALL
01

Validate

envelo validate

Pre-flight check: credentials, API connectivity, source mappings, fail-closed mode.

02

Start

envelo start -d

Starts the enforcement agent as a background daemon. For auto-start: envelo service install

03

Begin CAT-72

Go to app.sentinelauthority.org → your case → CAT-72 tab → click Begin Test. The 72-hour timer starts immediately.

The Agent Does Everything

Every operation — monitoring, diagnostics, infrastructure, boundaries — is a single command.

Running

SECURITY MODEL
Zero Inbound. Zero Trust.
Outbound only — ENVELO opens zero inbound ports on your network
No remote access — Sentinel cannot access your system, network, or data
No commands from cloud — One-way: telemetry out, nothing in
No PHI/PII — Only ODD parameter values transmitted. No business data.
Signed telemetry — Cryptographically signed and hash-chain linked
Fail-closed — Connection drops → agent blocks all actions (safe state)
Certificate pinning — TLS 1.3 with pinning prevents MITM interception
SYSTEM REQUIREMENTS
OSLinux (Ubuntu 20.04+, RHEL 8+), macOS 12+, Windows Server 2019+
Python3.7 or higher
CPU2 cores minimum

NetworkOutbound HTTPS (port 443)
ClockNTP-synchronized (±1 second)
