CONFORMANCE PROCEDURE

CAT-72

Convergence Authorization Test — 72 hours of continuous proof.

No waivers. No shortcuts. 72 hours of evidence or no conformance.

01

Continuous Demo

72-hour sustained operation within declared ODD bounds. No interruptions, no resets.

02

Stress Testing

Multi-regime operational testing across edge conditions and state transitions.

03

Fail-Closed Proof

Verified halt behavior when ENVELO detects material deviation outside ODD.

Evidence Requirements

Cryptographically signed telemetry log
Hash-chain linked state recordings
Interlock activation events with timestamps
Convergence metrics and boundary proximity

Tolerance Declaration

Applicants declare operational tolerances prior to CAT-72. Tolerances are domain-specific based on:

  • Equipment manufacturer specifications
  • Applicable regulatory requirements
  • Risk profile and consequence severity
  • Industry standards and best practices

Must be quantitative, bounded, documented. Cannot be modified during test.

RUNTIME ENFORCEMENT

The Interlock

Every action passes through. No exceptions. No bypasses.

Non-bypassable. Fail-closed. Every action validated before execution.

INTERCEPT

Capture Signal

Every model output is routed through the interlock. No direct path to actuators exists.

VALIDATE

Check Bounds

Action evaluated against declared ODD constraints. Within bounds? Permit. Outside? Block.

ENFORCE

Act or Halt

Permitted actions execute. Violations trigger fail-closed halt. Everything logged.

The Non-Bypass Guarantee

The interlock sits between decision and action — architecturally. There is no code path, no override flag, no emergency bypass that allows an action to reach actuators without validation. This isn't policy. It's physics.

✓ Permit in-bounds ✗ Block out-of-bounds ◉ Log everything

SECURITY ARCHITECTURE

Zero Trust, Zero Access

Enterprise security teams ask: "Is this agent a backdoor into our systems?" The answer is architecturally impossible. The ENVELO Agent uses one-way data flow that makes remote access or control physically impossible.

One-way data architecture. No inbound connections. No remote access.

01

Zero Listening Ports

Agent opens no inbound ports. Only outbound HTTPS to api.sentinelauthority.org.

02

Customer Controlled

You deploy, configure, and terminate. We cannot start, stop, or modify remotely.

03

TLS 1.3 + Pinning

All communications encrypted with certificate pinning. Data encrypted at rest.

The Core Guarantee

Sentinel Authority cannot send commands to your system, execute code on your infrastructure, access your network, or control your autonomous systems. The agent only observes and reports—it has no actuator access.

✗ No Remote Commands ✗ No Code Execution ✗ No Network Access ✗ No Actuator Control
